# 📝 คู่มือการส่งสอบ และวิธีแก้ปัญหา Docker ขั้นสุดท้าย

ตอนนี้ตัวโค้ดและใบงาน `Exam-Result.md` ถูกแก้ไขและตั้งค่าไว้ให้เสร็จสมบูรณ์ 100% แล้ว สิ่งเดียวที่เหลืออยู่คือ **การทำให้โปรแกรมรันขึ้นมาได้บนเครื่องของคุณ เพื่อถ่ายภาพหน้าจอ (Screenshots)** ไปประกอบในรายงานก่อนส่งครับ

ปัญหาที่คุณกำลังเจอคือ `failed to connect to the docker API` ซึ่งเกิดจากโปรแกรม **Docker Desktop** ในเครื่องของคุณค้างหรือปิดตัวไปอย่างไม่ถูกต้อง ทำให้ไม่สามารถใช้งานคำสั่ง `docker` ได้

กรุณาทำตามขั้นตอนด้านล่างนี้ทีละสเต็ปครับ:

---

## 🛠️ Step 1: วิธีแก้ปัญหา Docker ค้าง / ไม่ทำงาน

1. **ปิด Docker Desktop ตัวเก่าที่ค้างอยู่ให้สนิท:**
   - กดปุ่ม `Ctrl + Shift + Esc` เพื่อเปิด **Task Manager**
   - มองหาโปรเซสที่ชื่อว่า `Docker Desktop` หรือ `com.docker.backend`
   - คลิกขวาแล้วเลือก **End Task** (ปิดให้หมดทุกตัวที่เกี่ยวข้องกับ Docker)
   
2. **เปิด Docker Desktop ขึ้นมาใหม่:**
   - ไปที่เมนู **Start** (กดปุ่ม Windows)
   - พิมพ์ค้นหาคำว่า **Docker Desktop** แล้วกด **Enter** เพื่อเปิดโปรแกรม
   - **รอประมาณ 1 - 2 นาที** จนกว่ามุมซ้ายล่างของหน้าต่าง Docker Desktop จะเปลี่ยนเป็นสีเขียวและขึ้นคำว่า `Engine running` (หรือดูไอคอนรูปปลาวาฬที่มุมขวาล่างของจอคอมพิวเตอร์ ต้องหยุดขยับและขึ้นว่า Docker is running)

> **สำคัญมาก:** ห้ามพิมพ์คำสั่ง `docker compose` ใดๆ จนกว่าจะมั่นใจว่า Docker Desktop ขึ้นสถานะ **Engine running (สีเขียว)** เรียบร้อยแล้ว ไม่เช่นนั้นจะเจอ Error หาไฟล์ไม่เจออีกครับ

---

## 🚀 Step 2: รันโปรเจกต์ (Local Staging)

เมื่อ Docker รันเสร็จสมบูรณ์แล้ว ให้คุณกลับมาที่ Terminal ของ VSCode และพิมพ์คำสั่งนี้:

```bash
docker compose up --build -d
```

รอจนกว่าระบบจะดาวน์โหลดและสร้างคอนเทนเนอร์เสร็จ (ถ้าขึ้นสถานะ `Started` หรือ `Healthy` สีเขียวทั้งหมด แปลว่ารันสำเร็จ)
